## 1. 产品概述

一个模拟"老校内网/人人网"风格的社交系统原型，用同一批本地模拟用户演示好友关系、动态、照片和隐私可见性的联动效果。用户可切换不同模拟账号，体验自己、好友、非好友看到的内容范围差异。

- 目标用户：开发者/产品经理，用于快速验证社交系统核心业务骨架
- 核心价值：以可操作的界面展示社交系统的权限联动逻辑，而非静态说明页

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 当前登录用户 | 通过顶部切换器选择 | 按好友关系查看不同范围的动态和照片 |
| 好友用户 | 与当前用户互为好友 | 可查看"好友可见"内容 |
| 非好友用户 | 与当前用户无好友关系 | 仅可查看"公开"内容 |

### 2.2 功能模块

1. **首页/动态流页**：顶部统计栏、动态发布区、动态照片流、评论回复区
2. **个人主页**：用户详情、好友列表、照片墙、动态列表
3. **搜索/加好友页**：搜索同学、发起加好友请求

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 动态流页 | 顶部统计栏 | 显示好友数、动态数、照片数，随操作实时变化 |
| 动态流页 | 账号切换器 | 下拉切换当前模拟账号，页面内容随权限联动刷新 |
| 动态流页 | 动态发布区 | 发布带照片占位的动态，可选可见性（公开/好友可见/仅自己） |
| 动态流页 | 动态照片流 | 按可见性过滤后展示动态列表，含照片缩略图、点赞、评论入口 |
| 动态流页 | 评论回复区 | 对动态进行评论、对评论进行回复 |
| 个人主页 | 用户信息卡片 | 头像、姓名、学校/班级、好友数、签名 |
| 个人主页 | 好友列表 | 展示该用户的好友，支持点击跳转 |
| 个人主页 | 照片墙 | 按可见性展示该用户的照片，支持切换可见性 |

| 搜索加好友 | 搜索栏 | 按姓名/学校搜索同学 |
| 搜索加好友 | 用户卡片 | 展示搜索结果，显示好友状态，发起加好友请求 |

## 3. 核心流程

### 3.1 账号切换与可见性联动

用户通过顶部切换器选择模拟账号 → 系统根据当前账号与各内容所有者的好友关系 → 过滤动态和照片的可见范围 → 页面实时刷新展示

### 3.2 发布动态与可见性控制

用户在发布区输入内容 → 可选上传照片占位 → 选择可见性（公开/好友可见/仅自己） → 动态出现在流中 → 其他账号根据关系看到或看不到

### 3.3 搜索加好友

输入关键词 → 匹配模拟用户 → 点击"加好友" → 好友关系建立 → 顶部统计数字变化 → 切换到该好友主页可看到更多内容

### 3.4 评论回复

点击动态的评论按钮 → 展开评论区 → 输入评论 → 可对已有评论回复 → 评论实时显示

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：校内网经典蓝 (#3B5998) + 白色背景，辅以浅灰 (#F0F2F5)
- **按钮风格**：圆角按钮，主操作用蓝色填充，次操作用白底蓝边
- **字体**：标题用 16-18px 加粗，正文 14px 常规，辅助文字 12px 浅灰
- **布局风格**：左侧导航 + 中间内容流 + 右侧边栏，经典三栏布局
- **图标风格**：lucide-react 线性图标，保持简洁
- **整体氛围**：怀旧校内网风格，蓝色顶栏、白色卡片、简洁表格线

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 要素 |
|----------|----------|---------|
| 动态流页 | 顶部导航 | 蓝色背景、Logo、账号切换下拉、统计徽章 |
| 动态流页 | 左侧边栏 | 导航菜单、好友/同学列表缩略 |
| 动态流页 | 中间动态流 | 白色卡片、发布框、动态列表、评论区 |
| 动态流页 | 右侧边栏 | 推荐同学、快捷入口 |
| 个人主页 | 封面区 | 蓝色渐变背景、头像、基本信息 |
| 个人主页 | 内容区 | 标签切换（动态/照片/好友） |
| 搜索页 | 搜索栏 | 居中搜索框、结果卡片列表 |

### 4.3 响应式

- 桌面优先设计（最小宽度 1024px）
- 三栏布局在窄屏时侧边栏隐藏
- 触摸友好的按钮和交互区域
